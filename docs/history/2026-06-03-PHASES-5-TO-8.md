### Part B — Launcher Generator CI Fix (was D1)

**Problem:** PortableApps.com CDN rejects TLS from GitHub-hosted runners.
CI falls back to `--skip-launcher`, producing zips without `PASkillsPortable.exe`.

**Solution:** Self-host the extracted Launcher Generator in the repo.

| # | Task | Detail |
|---|------|--------|
| 5B-1 | Download + extract locally | Download `PortableApps.comLauncher_2.2.4.paf.exe`, extract with `/S /D=<scratch>`, identify minimum file set for `PortableApps.comLauncherGenerator.exe` to run |
| 5B-2 | Measure + decide git vs LFS | If extracted ≤ ~10 MB → plain git under `bundling/launcher-gen/`. If larger → add LFS pattern for `bundling/launcher-gen/**` to `.gitattributes` |
| 5B-3 | Commit extracted tree | `bundling/launcher-gen/2.2.4/` with a README documenting upstream URL, version, SHA-256, license (GPL-v2), re-vendor procedure |
| 5B-4 | Wire into build.py | Add `bundling/launcher-gen/` to `_find_launcher_generator()`'s search hints. Both local and CI builds find it without flags |
| 5B-5 | Simplify release.yml | Delete "Install PortableApps.com Launcher Generator" step, delete `SKIP_LAUNCHER` / fallback-zip plumbing, simplify build invocation, remove conditional release-body text |
| 5B-6 | Update NOTICE + CHANGELOG | Credit PortableApps.com Launcher (GPL-v2), link upstream source |
| 5B-7 | Verify on CI | Push tag `v0.4.1-rc1` (prerelease). Pass criteria: workflow green, generator invoked, zip contains `PASkillsPortable.exe` at root, fallback step skipped. Delete RC after verification. |
